# Realtime Fraud Alerts

Dashboard для мониторинга подозрительных транзакций в реальном времени. Push-based архитектура через WebSocket.

## Запуск

```bash
docker-compose up
```

Открой http://localhost:8000

## Как работает

1. Backend генерирует случайные транзакции каждые 1-5 секунд
2. Правила детекции проверяют каждую транзакцию
3. Подозрительные помечаются и подсвечиваются в UI
4. WebSocket доставляет события мгновенно

## Design decisions

### Почему WebSocket, а не polling/SSE
- Polling: задержка между опросами, лишний трафик
- SSE: односторонний, нельзя отправлять от клиента
- WebSocket: двусторонний, низкая задержка, можно расширить

### Почему батчинг обновлений
При быстром потоке событий (10+/сек) ре-рендер DOM на каждое событие вызывает лаги. Батчинг раз в 300мс снижает нагрузку в ~30x без заметной потери отзывчивости.

### Почему простые правила, а не ML
- Правила работают за O(1), ML требует inference pipeline
- Для demo достаточно threshold-based подхода
- В production нужна модель — но это pet-проект

## Known limitations

- Правила простые (порог + частота) — дают false positive
- Нет persistent storage — при перезапуске теряется история
- Один инстанс — нет горизонтального масштабирования
